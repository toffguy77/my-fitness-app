# Аутентификация и Онбординг

## Экраны в Разделе

1. **Start Screen** (Стартовый экран)
2. **Registration** (Регистрация)
3. **Authentication** (Вход в систему)
4. **Password Reset** (Восстановление пароля)
5. **Onboarding: Photo Upload** (Первичная загрузка фото)
6. **Onboarding: Locality** (Выбор региона)
7. **Onboarding: Social** (Социальные настройки)
8. **Onboarding: Apple Health** (Интеграция с Apple Health)

## 1. Start Screen (Стартовый экран)

### Назначение
Первый экран при запуске приложения для новых пользователей или после выхода из системы.

### UI Компоненты
- **Логотип приложения**: Центрально расположенный брендинг
- **Слоган/Описание**: Краткое value proposition (1-2 предложения)
- **Кнопка "Войти"**: Для существующих пользователей
- **Кнопка "Зарегистрироваться"**: Для новых пользователей
- **Язык интерфейса**: Переключатель языка (опционально)

### Поведение
- При первом запуске показывается автоматически
- Доступен через "Выход" из основного приложения
- Поддержка анимации при появлении (fade-in)

### Связи с Экранами
- **→ Registration**: При нажатии "Зарегистрироваться"
- **→ Authentication**: При нажатии "Войти"

### Бизнес-Правила
- Если пользователь уже авторизован (токен валиден), автоматический переход на Dashboard
- Проверка версии приложения и принудительное обновление при необходимости

### Метрики
- Количество переходов на регистрацию vs вход
- Показатель отказов (bounce rate) на первом экране

## 2. Registration (Регистрация)

### Назначение
Создание нового аккаунта пользователя.

### UI Компоненты

#### Форма Регистрации
- **Email**: Поле ввода с валидацией формата
- **Пароль**: Поле с индикатором сложности пароля
- **Подтверждение пароля**: Поле с валидацией совпадения
- **Имя**: Опциональное поле (может заполняться позже)

#### Дополнительно
- **Чекбокс**: Согласие с условиями использования и политикой конфиденциальности
- **Ссылки**: На документы Terms & Privacy Policy
- **Кнопка "Создать аккаунт"**: Основное действие
- **Текст**: "Уже есть аккаунт? Войти" → переход на Authentication

### Валидация

#### Email
```
- Формат: name@domain.com
- Уникальность: проверка в базе данных
- Верификация: отправка письма подтверждения
```

#### Пароль
```
Требования:
- Минимум 8 символов
- Минимум 1 заглавная буква
- Минимум 1 цифра
- Минимум 1 специальный символ
```

### Процесс Регистрации

1. **Ввод данных** → Валидация на фронтенде
2. **Отправка на сервер** → Создание аккаунта
3. **Email верификация** → Отправка письма со ссылкой
4. **Подтверждение** → Переход на онбординг или дашборд

### Обработка Ошибок
- **Email занят**: "Этот email уже используется. Войти?"
- **Слабый пароль**: "Пароль не соответствует требованиям безопасности"
- **Сетевая ошибка**: "Проверьте подключение к интернету"

### Связи с Экранами
- **← Start Screen**: Возврат назад
- **→ Onboarding: Photo Upload**: После успешной регистрации
- **→ Authentication**: При клике "Уже есть аккаунт"

### Бизнес-Правила
- Обязательная email-верификация перед доступом к функционалу
- Возможность регистрации через соцсети (опционально): Apple ID, Google
- Автоматическое создание профиля пользователя с дефолтными настройками

### Метрики
- Conversion rate (Start → Registration → Success)
- Время заполнения формы
- Количество ошибок валидации
- % отказов на каждом поле формы

## 3. Authentication (Вход в систему)

### Назначение
Аутентификация существующих пользователей.

### UI Компоненты
- **Email/Username**: Поле ввода
- **Пароль**: Скрытое поле с опцией "Показать пароль"
- **Чекбокс "Запомнить меня"**: Для автоматического входа
- **Кнопка "Войти"**: Основное действие
- **Ссылка "Забыли пароль?"**: → Password Reset
- **Текст**: "Нет аккаунта? Зарегистрироваться" → Registration

### Процесс Входа

1. **Ввод учетных данных** → Валидация формата
2. **Отправка на сервер** → Проверка credentials
3. **Получение токена** → Сохранение в secure storage
4. **Переход на Dashboard** → Загрузка данных пользователя

### Обработка Ошибок
- **Неверные данные**: "Неправильный email или пароль"
- **Email не верифицирован**: "Подтвердите email. Отправить письмо повторно?"
- **Аккаунт заблокирован**: "Ваш аккаунт временно заблокирован. Обратитесь в поддержку"
- **Слишком много попыток**: "Превышен лимит попыток входа. Попробуйте через X минут"

### Связи с Экранами
- **← Start Screen**: Возврат назад
- **→ Dashboard**: При успешной аутентификации
- **→ Password Reset**: При клике "Забыли пароль"
- **→ Registration**: При клике "Зарегистрироваться"

### Бизнес-Правила
- Максимум 5 неудачных попыток входа → временная блокировка (15 минут)
- Сессия действительна 30 дней при активации "Запомнить меня"
- Логирование всех попыток входа для безопасности
- Двухфакторная аутентификация (опционально)

### Альтернативные Методы Входа
- **Apple ID**: Sign in with Apple
- **Google**: Google OAuth
- **Биометрия**: Face ID / Touch ID (после первого входа)

### Метрики
- Success rate (успешных входов)
- Количество использования "Забыли пароль?"
- % входов через соцсети vs email
- Время от входа до первого действия в приложении

## 4. Password Reset (Восстановление пароля)

### Назначение
Восстановление доступа при забытом пароле.

### Шаг 1: Запрос Сброса

#### UI Компоненты
- **Email**: Поле ввода email аккаунта
- **Кнопка "Отправить инструкции"**: Запрос на сброс
- **Текст**: Пояснение процесса восстановления
- **Ссылка "Вспомнили пароль? Войти"**: → Authentication

#### Поведение
1. Пользователь вводит email
2. Система отправляет письмо со ссылкой для сброса
3. Показывается подтверждающее сообщение

### Шаг 2: Email с Инструкциями

```
Тема: Восстановление пароля

Привет [Имя],

Мы получили запрос на сброс пароля для вашего аккаунта.

[Кнопка: Сбросить пароль]

Или скопируйте ссылку: https://app.example.com/reset/[TOKEN]

Ссылка действительна 24 часа.

Если вы не запрашивали сброс пароля, проигнорируйте это письмо.
```

### Шаг 3: Установка Нового Пароля

#### UI Компоненты
- **Новый пароль**: Поле с индикатором сложности
- **Подтверждение пароля**: Поле валидации
- **Кнопка "Сохранить новый пароль"**: Подтверждение
- **Индикатор сложности пароля**: Визуальная шкала

#### Поведение
1. Валидация сложности пароля
2. Проверка совпадения полей
3. Обновление пароля в БД
4. Автоматический вход → Dashboard

### Безопасность
- **Токен**: Одноразовый, действителен 24 часа
- **История паролей**: Нельзя использовать последние 5 паролей
- **Уведомление**: Email о смене пароля на основной адрес

### Связи с Экранами
- **← Authentication**: Возврат к входу
- **→ Dashboard**: После успешного сброса и входа

### Обработка Ошибок
- **Email не найден**: "Аккаунт с таким email не существует"
- **Токен истек**: "Ссылка устарела. Запросить новую?"
- **Слабый пароль**: "Пароль не соответствует требованиям"

### Метрики
- Количество запросов на сброс пароля
- Conversion rate (запрос → успешный сброс)
- Время от запроса до сброса
- % пользователей, завершивших сброс

## 5. Onboarding: Photo Upload

### Назначение
Первичная загрузка фото пользователя для отслеживания прогресса.

### UI Компоненты
- **Заголовок**: "Загрузите свое первое фото"
- **Описание**: Зачем это нужно, как будет использоваться
- **Область загрузки**: Drag & drop или выбор файла
- **Опции**:
  - Сделать фото сейчас (камера)
  - Выбрать из галереи
  - Пропустить (можно загрузить позже)
- **Рекомендации**: "Желательно фото в полный рост, хорошее освещение"
- **Кнопка "Продолжить"**: Переход к следующему шагу
- **Прогресс-бар**: "Шаг 1 из 4" онбординга

### Поведение
- Загруженное фото сохраняется как "Фото 1" с датой
- Превью загруженного фото перед сохранением
- Возможность crop/rotate фото
- Автоматическое сжатие для оптимизации размера

### Хранение Фото
```
Формат: JPEG
Максимальный размер: 5MB (оригинал)
Thumbnail: 300x300px для быстрой загрузки
Метаданные: дата, тип (фронт/профиль/бэк), вес на момент фото
```

### Связи с Экранами
- **← Registration**: Первый экран после регистрации
- **→ Onboarding: Locality**: Следующий шаг онбординга
- **→ Dashboard**: Если пропущен весь онбординг

### Бизнес-Правила
- Фото опционально (можно пропустить)
- Рекомендуется загружать каждую неделю для отслеживания прогресса
- Фото видны только пользователю и его куратору

### Метрики
- % пользователей, загружающих фото на онбординге
- Среднее количество попыток перед успешной загрузкой
- % пользователей, пропускающих этот шаг

## 6. Onboarding: Locality (Регион)

### Назначение
Настройка региональных параметров (язык, часовой пояс, единицы измерения).

### UI Компоненты
- **Заголовок**: "Настройте региональные параметры"
- **Язык интерфейса**: Dropdown (Русский, English, etc.)
- **Часовой пояс**: Автоопределение или ручной выбор
- **Единицы измерения**:
  - Вес: кг / lb
  - Рост: см / футы и дюймы
  - Расстояние: км / мили
  - Калории: ккал / кДж
- **Кнопка "Продолжить"**: Сохранить и перейти далее
- **Прогресс-бар**: "Шаг 2 из 4"

### Автоопределение
- Определение по IP-адресу или геолокации
- Предложение настроек по умолчанию для региона
- Возможность ручной корректировки

### Связи с Экранами
- **← Onboarding: Photo Upload**: Предыдущий шаг
- **→ Onboarding: Social**: Следующий шаг
- **→ Dashboard**: Если пропущен остаток онбординга

### Бизнес-Правила
- Язык интерфейса применяется немедленно
- Часовой пояс влияет на отображение времени логов
- Единицы измерения используются во всем приложении

### Метрики
- Распределение пользователей по языкам
- Распределение по часовым поясам
- % изменений дефолтных настроек

## 7. Onboarding: Social (Социальные настройки)

### Назначение
Настройка приватности и видимости профиля.

### UI Компоненты
- **Заголовок**: "Настройка приватности"
- **Публичный профиль**: Toggle (Вкл/Выкл)
  - Вкл: Профиль виден другим пользователям
  - Выкл: Профиль скрыт, виден только куратору
- **Показывать прогресс**:
  - Всем пользователям
  - Только друзьям
  - Никому
- **Разрешить комментарии**: Toggle
- **Разрешить личные сообщения**: Toggle
- **Кнопка "Продолжить"**: Сохранить и перейти далее
- **Прогресс-бар**: "Шаг 3 из 4"

### Дефолтные Настройки
```
Публичный профиль: Выкл
Показывать прогресс: Никому
Комментарии: Выкл
Личные сообщения: Вкл (для куратора)
```

### Связи с Экранами
- **← Onboarding: Locality**: Предыдущий шаг
- **→ Onboarding: Apple Health**: Следующий шаг
- **→ Dashboard**: Если пропущен последний шаг

### Бизнес-Правила
- Настройки можно изменить позже в профиле
- Куратор всегда имеет доступ к данным пользователя
- При публичном профиле применяются дополнительные модерационные правила

### Метрики
- % пользователей с публичными профилями
- Распределение настроек приватности
- Корреляция публичности и engagement

## 8. Onboarding: Apple Health Integration

### Назначение
Интеграция с Apple Health для автоматического импорта и экспорта данных.

### UI Компоненты
- **Заголовок**: "Подключить Apple Health?"
- **Описание**: Преимущества интеграции
  - Автоматический импорт шагов, сна, ЧСС
  - Экспорт тренировок и питания
  - Централизованное хранение всех health-данных
- **Список разрешений**:
  - [✓] Шаги
  - [✓] Активные калории
  - [✓] Вес
  - [✓] Тренировки
  - [✓] Сон (если доступно)
  - [✓] ЧСС (если доступно)
- **Кнопка "Подключить Apple Health"**: Запрос разрешений
- **Кнопка "Пропустить"**: Продолжить без интеграции
- **Прогресс-бар**: "Шаг 4 из 4"

### Процесс Интеграции

1. **Нажатие "Подключить"** → Системный диалог Apple Health
2. **Выбор разрешений** → Пользователь гранулярно выбирает, что разрешить
3. **Подтверждение** → Приложение получает доступ
4. **Первичная синхронизация** → Импорт данных за последние 7 дней
5. **Переход на Dashboard** → Завершение онбординга

### Разрешения Apple Health

#### Чтение (Read)
- Шаги
- Активные калории
- Вес
- Рост
- Дата рождения
- Биологический пол
- История тренировок
- Сон
- Пульс

#### Запись (Write)
- Вес
- Тренировки (кардио, силовые)
- Активные калории
- Питание (КБЖУ)

### Синхронизация
- **Частота**: Каждый час при активном приложении
- **Приоритет**: Данные из приложения имеют приоритет при конфликтах
- **История**: Импорт данных за последние 30 дней при первом подключении

### Связи с Экранами
- **← Onboarding: Social**: Предыдущий шаг
- **→ Dashboard**: Завершение онбординга (всегда, независимо от подключения)

### Обработка Ошибок
- **Отказ в разрешениях**: "Вы сможете подключить Apple Health позже в настройках"
- **Ошибка синхронизации**: "Не удалось синхронизировать данные. Повторить?"

### Бизнес-Правила
- Интеграция опциональна
- Можно подключить/отключить в любой момент в настройках
- Отключение не удаляет данные из приложения
- Данные из Apple Health имеют метку "импортировано" для различения

### Метрики
- % пользователей iOS, подключающих Apple Health
- % пользователей, использующих автоимпорт шагов/веса
- Объем синхронизированных данных

## Общие Элементы Онбординга

### Навигация
- **Прогресс-бар**: Визуальный индикатор текущего шага (1/4, 2/4, 3/4, 4/4)
- **Кнопка "Назад"**: Возврат на предыдущий шаг
- **Кнопка "Пропустить"**: Переход сразу на Dashboard (любой шаг)

### Состояние Прохождения
```javascript
Onboarding State:
{
  step: 1-4,
  photoUploaded: boolean,
  localitySet: boolean,
  socialConfigured: boolean,
  appleHealthConnected: boolean,
  completed: boolean
}
```

### После Завершения Онбординга
- Сохранение флага `onboardingCompleted: true` в профиле
- Автоматический редирект на Dashboard
- Возможность перепройти онбординг через настройки

### Tips & Best Practices
- **Краткость**: Каждый шаг должен быть понятен за 5-10 секунд
- **Визуализация**: Использовать иллюстрации для объяснения преимуществ
- **Прогресс**: Четко показывать, сколько осталось шагов
- **Flexibility**: Возможность пропустить или вернуться позже

## Безопасность и Конфиденциальность

### Хранение Паролей
- **Хеширование**: bcrypt с salt
- **Транспорт**: Только HTTPS
- **Никогда**: Не логировать и не показывать пароли в открытом виде

### Токены Аутентификации
- **Формат**: JWT (JSON Web Token)
- **Срок действия**: 30 дней (с "Запомнить меня"), 7 дней (без)
- **Refresh tokens**: Автоматическое обновление при активности
- **Хранение**: Secure storage (Keychain на iOS, Encrypted Shared Preferences на Android)

### Email Верификация
- **Обязательная**: Для доступа к полному функционалу
- **Повторная отправка**: Доступна каждые 60 секунд
- **Токен**: Одноразовый, действителен 24 часа

### GDPR Compliance
- **Согласие**: Явное согласие на обработку данных при регистрации
- **Прозрачность**: Ссылки на Privacy Policy и Terms of Service
- **Право на удаление**: Возможность полного удаления аккаунта
- **Экспорт данных**: Возможность скачать все свои данные

## Аналитика и Метрики

### Воронка Регистрации
```
Start Screen → Registration → Email Verification → Onboarding → Dashboard
```

**Ключевые метрики**:
- Drop-off rate на каждом шаге
- Время прохождения каждого шага
- Conversion rate (полное завершение)
