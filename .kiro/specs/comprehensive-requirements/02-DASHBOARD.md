# Главный Дашборд

## Обзор

Дашборд — это центральный экран приложения, где пользователь видит сводку всех ключевых метрик за выбранный день и может быстро перейти к любому разделу для детального просмотра или добавления данных.

---

## Структура Дашборда

### Основные Элементы

1. **Заголовок и Навигация**
2. **Календарная Навигация**
3. **Блоки Метрик** (виджеты)
4. **Нижняя Навигация** (Bottom Tab Bar)

---

## 1. Заголовок и Навигация

### UI Компоненты

- **Логотип/Название приложения**: В левом верхнем углу
- **Аватар пользователя**: В правом верхнем углу → переход в Профиль
- **Иконка уведомлений**: Бейдж с количеством непрочитанных → раздел Notifications
- **Текущая дата**: Подзаголовок "Сегодня, 11 февраля 2026" или выбранная дата

### Связи
- **→ Profile**: При клике на аватар
- **→ Notifications**: При клике на иконку уведомлений

---

## 2. Календарная Навигация

### Назначение
Переключение между днями для просмотра исторических данных или планирования будущего.

### UI Компоненты

- **Стрелка влево**: Предыдущий день
- **Текущая дата**: Большой текст по центру (например, "ПН, 10 Фев")
- **Стрелка вправо**: Следующий день
- **Кнопка "Сегодня"**: Быстрый переход к текущей дате
- **Календарь (опционально)**: При клике на дату — развернутый календарь для выбора произвольной даты

### Поведение

- **Свайп**: Горизонтальный свайп влево/вправо для переключения дней
- **Анимация**: Плавная смена содержимого при переключении
- **Индикаторы**: Точки или цвета на календаре для дней с неполными данными

### Состояния Дня

```
✅ Все метрики заполнены
⚠️ Частично заполнен (есть незаполненные метрики)
❌ Пустой день
📅 Будущий день (планирование)
```

### Бизнес-Правила

- По умолчанию открывается текущий день
- Можно просматривать данные за любую дату
- Для будущих дней показываются плановые значения (если заполнены)
- Макс��мальная глубина истории: 2 года назад

## 3. Блоки Метрик (Виджеты)

Каждый блок — это модульный виджет с кратким отображением метрики и возможностью перехода к детальному экрану.

### 3.1 Блок "Вес" (Weight Block)

#### Отображение
- **Заголовок**: "Вес"
- **Текущее значение**: 75.3 кг
- **Изменение**: +0.2 кг за неделю (стрелка вверх/вниз)
- **Мини-график**: Тренд за последние 7 дней
- **Бейдж внимания**: ⚠️ если вес не логировался >3 дней

#### Действия
- **Клик на виджет**: → Модальное окно ввода веса
- **Long press**: → График веса с историей

#### Связи
- **→ Weight Input Modal**: Быстрый ввод
- **→ Progress Section (Weight Graph)**: Детальный график

---

### 3.2 Блок "Питание" (Nutrition Block)

#### Отображение
- **Заголовок**: "Питание"
- **Круговой прогресс-бар**: % от дневной нормы калорий
  - Зеленый: 80-100%
  - Желтый: 50-79%
  - Красный: <50% или >100%
- **Центр круга**:
  - Текущие: 1850 ккал
  - Норма: 2000 ккал
- **Макросы** (под кругом):
  - Белки: 120г / 150г
  - Жиры: 50г / 67г
  - Углеводы: 180г / 250г
- **Бейдж внимания**: ⚠️ если питание не логировалось сегодня

#### Действия
- **Клик на виджет**: → Food Tracker (главный экран дневника питания)
- **Кнопка "+"**: → Быстрое добавление продукта

#### Связи
- **→ Food Tracker**: Полный дневник питания
- **→ Quick Add Food**: Модальное окно добавления

---

### 3.3 Блок "Тренировка" (Workout Block)

#### Отображение
- **Заголовок**: "Тренировка"
- **Статус**:
  - ✅ "Завершено" (если тренировка залогирована)
  - ❌ "Не выполнено" (если план есть, но не залогирована)
  - ➕ "Добавить тренировку" (если плана нет)
- **Детали** (если завершена):
  - Тип: Силовая / Кардио / Йога и т.д.
  - Длительность: 60 мин
  - Нагрузка: Средняя
  - Самочувствие: 😊 Хорошо
- **Бейдж внимания**: ⚠️ если тренировка запланирована, но не выполнена

#### Действия
- **Клик**: → Форма добавления/редактирования тренировки
- **Quick Actions**:
  - Начать тренировку (запуск таймера)
  - Отметить как выполненную
  - Пропустить

#### Связи
- **→ Workout Form**: Детальная форма
- **→ Workout Timer**: Встроенный таймер тренировки

---

   - Бейдж: количество непрочитанных сообщений

- Двойной клик на активную вкладку: scroll to top

## Attention Indicators (Бейджи Внимания)

### Назначение
Привлечь внимание пользователя к метрикам, требующим действия.

### Типы Бейджей

1. **⚠️ Желтый**: Рекомендация обратить внимание
2. **🔴 Красный**: Критично (например, вес не логировался >7 дней)
3. **🔵 Синий**: Новая информация (новая задача от куратора)

### Логика Появления

#### Питание
- Не залогировано до 12:00 → Желтый
- Не залогировано весь день → Красный

#### Вес
- Не логировался >3 дней → Желтый
- Не логировался >7 дней → Красный

#### Тренировка
- Запланирована, но не выполнена → Желтый
- Пропущено 2+ тренировки подряд → Красный

#### Фото
- Не загружалось >7 дней → Желтый
- Не загружалось >14 дней → Красный

#### Задачи
- <50% выполнено к концу дня → Желтый

### Принципы
- Не перегружать пользователя уведомлениями
- Максимум 2-3 бейджа одновременно
- Приоритет критическим метрикам

## Состояния и Поведение

### Загрузка Дашборда

```
1. Показать skeleton/placeholder блоков
2. Загрузить данные из локального кэша (мгновенно)
3. Запросить актуальные данные с сервера
4. Обновить блоки при получении данных
```

### Pull-to-Refresh
- Свайп вниз → обновление всех данных
- Индикатор загрузки
- Анимация обновления блоков

### Пустое Состояние
Для нового пользователя или пустого дня:

```
"Давайте начнем! 🎯"
- Добавьте свой вес
- Залогируйте первый прием пищи
- Запланируйте тренировку
```

### Ошибки
- **Нет интернета**: "Offline mode. Данные будут синхронизированы позже"
- **Ошибка сервера**: "Не удалось загрузить данные. Повторить?"
- **Тайм-аут**: Автоматический retry через 3 сек

## Персонализация Дашборда

### Настраиваемые Элементы
- **Порядок блоков**: Drag & drop для изменения порядка
- **Видимость блоков**: Скрыть неиспользуемые блоки
- **Единицы измерения**: кг/lb, км/мили и т.д.

### Адаптивность
- **Цели**: Динамическое изменение целей (калории, вес) по плану куратора
- **Фокус**: Выделение приоритетных метрик на основе целей пользователя

---

## Технические Требования

### Производительность
- Загрузка дашборда: <1 сек
- Обновление блока: <200 мс
- Переключение дней: <300 мс (анимация)

### Кэширование
- Локальный кэш: последние 7 дней
- Офлайн-режим: полная функциональность просмотра и ввода данных
- Синхронизация: автоматическая при восстановлении связи

### Оптимизация
- Lazy loading изображений
- Виртуализация списков (для задач, если много)
- Throttling запросов к серверу
