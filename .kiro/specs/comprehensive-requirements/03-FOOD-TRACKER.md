# Трекер Питания (Food Tracker)

## Обзор

Трекер питания — один из ключевых модулей приложения, позволяющий пользователям отслеживать потребление калорий, макронутриентов (КБЖУ) и получать персонализированные рекомендации по питанию.

---

## Экраны в Разделе

1. **Food Tracker Main** (Дневник питания)
2. **Food Recommendations** (Рекомендации питания)
3. **Add Food: Methods Selection** (Выбор способа добавления)
4. **Add Food: Search** (Поиск по базе продуктов)
5. **Add Food: Barcode Scanner** (Сканирование штрих-кода)
6. **Add Food: AI Photo Recognition** (Распознавание еды на фото)
7. **Add Food: Chat with Curator** (Добавление через куратора)
8. **Food Details** (Детальная информация о продукте)

---

```javascript
function getRecommendations(user, currentIntake) {
  const remaining = {
    calories: user.dailyGoal.calories - currentIntake.calories,
    protein: user.dailyGoal.protein - currentIntake.protein,
    fat: user.dailyGoal.fat - currentIntake.fat,
    carbs: user.dailyGoal.carbs - currentIntake.carbs
  };

  // Приоритизация недостающих макросов
  const priority = getPriorityMacro(remaining);

  // Фильтрация по диетическим ограничениям
  const filtered = filterByDiet(user.dietRestrictions);

  // Ранжирование по соответствию целям
  const ranked = rankByGoalMatch(filtered, remaining, priority);

  return ranked.slice(0, 10); // Топ-10 рекомендаций
}
```

### Персонализация

#### Факторы Учета
1. **Текущий дефицит/профицит** макросов
2. **Время суток** (утро → сложные углеводы, вечер → легкие белки)
3. **История предпочтений** (часто логируемые продукты)
4. **Диетические ограничения** (аллергии, непереносимости)
5. **Сезонность** (сезонные овощи/фрукты)
6. **Бюджет** (если указан)

### Процесс Сканирования

1. **Открытие камеры** → Запрос разрешения (при первом использовании)
2. **Наведение на штрих-код** → Автоматическое распознавание
3. **Поиск в БД** → По номеру штрих-кода
4. **Результат**:
   - **Найден**: → Food Details с автозаполненными данными
   - **Не найден**: → Запрос на сервер Open Food Facts API
   - **Все еще не найден**: → Предложение ручного ввода или вопрос куратору

### Технические Требования

#### Поддерживаемые Форматы
- EAN-13 (европейские штрих-коды)
- UPC-A (американские штрих-коды)
- EAN-8
- Code 128

#### Производительность
- Время распознавания: <1 сек
- Работа в условиях плохого освещения (фонарик)
- Возможность загрузки фото штрих-кода из галереи

#### API Интеграция
```javascript
// Проверка в локальной БД
const product = await db.findByBarcode(barcode);

if (!product) {
  // Запрос к Open Food Facts
  const response = await fetch(
    `https://world.openfoodfacts.org/api/v0/product/${barcode}.json`
  );

  if (response.product) {
    // Конвертация в наш формат и сохранение
    const normalized = normalizeOpenFoodFactsData(response.product);
    await db.saveProduct(normalized);
    return normalized;
  }
}
```

#### 3. AI Модель (Backend)
```python
# Пример псевдокода обработки на бэкенде

def recognize_food_items(image):
    # Используем предобученную модель (например, Food-101, Custom CNN)
    model = load_model('food_recognition_v2')

    # Детекция объектов на изображении
    detected_items = model.detect(image)

    # Для каждого объекта определяем:
    # - Название продукта/блюда
    # - Уверенность распознавания (confidence)
    # - Примерный вес/объем

    results = []
    for item in detected_items:
        if item.confidence > 0.7:  # Порог уверенности
            food_info = lookup_nutrition(item.label)
            estimated_weight = estimate_portion_size(item)

            results.append({
                'name': item.label,
                'confidence': item.confidence,
                'weight': estimated_weight,
                'nutrition': calculate_nutrition(food_info, estimated_weight)
            })

    return results
```

### Обработка Результатов

#### Высокая Уверенность (>80%)
- Автоматическое заполнение продуктов
- Возможность корректировки веса

#### Средняя Уверенность (50-80%)
