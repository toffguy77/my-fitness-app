# Система Уведомлений (Notifications)

## Обзор

Комплексная система уведомлений для информирования пользователей о важных событиях, мотивации к действиям и социального взаимодействия.

---

## Экраны в Разделе

1. **Notifications List** (Список уведомлений)
2. **Notification Settings** (Настройки уведомлений)
3. **Notification Types** (Детали по типам)
   - Likes (Лайки)
   - Comments (Комментарии на контент)
   - Answers (Ответы на комментарии)
   - Content (Новый контент)
   - Thresholds (Пороговые значения)
   - Reminders (Напоминания)
   - Chat (Сообщения от куратора)
   - Offers (Специальные предложения)

---

// Маршрутизация в зависимости от типа

  case 'reminder':

## 3. Детальное Описание Типов Уведомлений

### 3.1 Likes (Лайки)

#### Триггер
Кто-то лайкнул:
- Пост пользователя
- Фото прогресса
- Комментарий пользователя

#### Формат Уведомления
```
❤️ Мария и еще 5 человек лайкнули ваше фото прогресса

[Посмотреть фото]
```

#### Группировка
```
Если несколько лайков за короткое время:
❤️ 12 человек лайкнули ваше фото

Вместо:
❤️ Мария лайкнула...
❤️ Иван лайкнул...
❤️ Петр лайкнул...
(×12)
```

#### Связь
**→ Content Detail**: Переход к залайканному контенту

### 3.5 Thresholds (Пороговые значения)

### Назначение
Уведомления при достижении или пересечении важных порогов.

#### Типы Пороговых Событий

**Вес**
```
⚠️ Вес не логировался 5 дней

Последнее измерение: 06 февраля (75.2 кг)

[Добавить вес сейчас] [Напомнить позже]
```

**Калории**
```
🔴 К 20:00 вы потребили только 1200 из 2000 ккал

Недостаток: -800 ккал

[Посмотреть рекомендации] [Уже поел]
```

**Макросы**
```
⚡ Осталось добавить 50г белка до нормы

Рекомендации:
- Куриная грудка (150г) +31г
- Творог (100г) +18г

[Добавить продукт]
```

**Тренировки**
```
⚠️ Пропущено 2 тренировки подряд

Запланированные:
- 09.02 - Силовая (не выполнена)
- 11.02 - Кардио (не выполнена)

[Добавить тренировку] [Изменить план]
```

**Вода**
```
💧 Вы выпили только 4 из 8 стаканов воды

[Отметить стакан] [Скрыть]
```

### Настройка Порогов

Пользователь может настроить:
- Значения порогов (например, напоминать если вес не логировался >X дней)
- Время проверки (например, проверять калории в 20:00)
- Тип уведомления (push / только в приложении)

#### Умная Логика
```javascript
// Не спамить, если пользователь уже в курсе
if (user.lastOpenedApp < 1hour && notification.type === 'reminder') {
  // Не отправлять, он же только что в приложении был
  skip();
}

// Адаптация к паттернам пользователя
if (user.usuallyLogsWeightOn === 'Monday, Friday' && today === 'Monday') {
  // Отправить напоминание утром понедельника
  schedule('07:00');
}
```

#### Связь
**→ [Соответствующий экран]**: Food Tracker, Weight Input, Workout и т.д.

### 3.6 Reminders (Напоминания)

#### Типы Напоминаний

**Регулярные (Расписание)**
```
Примеры:
- "Залогируйте завтрак" (ежедневно в 09:00)
- "Не забудьте взвеситься" (Пн, Ср, Пт в 07:00)
- "Время тренировки" (за 1 час до запланированной)
- "Выпейте стакан воды" (каждые 2 часа)
```

**Умные Напоминания**
```
На основе поведения пользователя:
- Если обычно логирует обед в 13:00-14:00, напомнить в 13:30
- Если пропускает завтраки, напомнить раньше (07:30)
- Если забывает фото, напомнить раз в неделю (Воскресенье)
```

**Мотивационные**
```
- "Вы на правильном пути! Осталось 2 тренировки до цели недели"
- "Отличная серия! 5 дней подряд в рамках нормы калорий"
- "До цели -10кг осталось всего 2кг!"
```

#### Частота
```javascript
const reminderFrequency = {
  'food_logging': {
    breakfast: 'daily',
    lunch: 'daily',
    dinner: 'daily'
  },
  'weight': {
    frequency: 'custom', // Monday, Wednesday, Friday
    time: '07:00'
  },
  'water': {
    interval: 120, // минут
    startTime: '09:00',
    endTime: '21:00'
  },
  'workout': {
    type: 'before_planned',
    advance: 60 // минут до запланированной
  }
};
```

#### Snooze (Отложить)
```
[Напомнить через...]
  15 минут
  30 минут
  1 час
  Сегодня вечером (19:00)
  Завтра утром (09:00)
```

#### Связь
**→ [Целевое действие]**: Прямой переход к действию (добавить еду, вес и т.д.)

## Умные Уведомления (Smart Notifications)

### Персонализация Времени

#### Machine Learning
```python
# Определение оптимального времени на основе истории
def get_optimal_reminder_time(user, action_type):
    historical_data = user.action_history[action_type]

    # Когда пользователь обычно выполняет это действие?
    common_times = analyze_patterns(historical_data)

    # Отправить за 30 мин до обычного времени
    return common_times.median - timedelta(minutes=30)
```

### Контекстная Релевантность

#### Геолокация (опционально)
```
Если включена геолокация:
- "Вы рядом с вашим спортзалом. Время тренировки!" (когда рядом с залом)
- "Вы дома. Может, приготовить полезный ужин?" (вечером дома)
```

#### Активность
```
Если интеграция с Apple Health:
- Обнаружена активность → автоматическое предложение залогировать тренировку
- Длительное отсутствие движения → "Время размяться!"
```

### Адаптивная Частота

```javascript
// Снижение частоты для активных пользователей
if (user.loggedDaysInRow > 14) {
  // Пользователь дисциплинирован, меньше напоминаний
  reduceReminderFrequency(0.5);
}

// Увеличение для неактивных
if (user.daysWithoutLogging > 3) {
  increaseReminderFrequency(1.5);
  addMotivationalMessage();
}
```

## Бизнес-Правила

### Opt-in vs Opt-out
- **Критичные** (куратор, здоровье): Opt-out (включены по умолчанию)
- **Социальные** (лайки, комментарии): Opt-out
- **Маркетинг** (акции, предложения): Opt-in (выключены по умолчанию)

### GDPR / Privacy
- Явное согласие на получение уведомлений
- Возможность полного отключения в любой момент
- Прозрачность: какие данные используются для персонализации

### Rate Limiting
```
Максимум уведомлений:
- 10 push/день (средний пользователь)
- 20 push/день (очень активный пользователь)
- Безлимит для критических (здоровье, куратор)
```

### Батареи Телефона
- Уважение к энергопотреблению
- Группировка фоновых проверок
- Использование эффективных API (iOS: BGTaskScheduler)

### Backend
```javascript
// Отправка уведомления
async function sendNotification(userId, notification) {
  const user = await User.findById(userId);

  // Проверка настроек пользователя
  if (!user.notificationSettings[notification.type].enabled) {
    return; // Пользователь отключил этот тип
  }

  // Проверка тихих часов
  if (isQuietHours(user.quietHours, notification.type)) {
    return; // Не беспокоить
  }

  // Отправка
  await pushService.send({
    userId: user.id,
    tokens: user.deviceTokens,
    title: notification.title,
    body: notification.body,
    data: notification.data,
    badge: user.unreadCount + 1
  });

  // Сохранение в БД
  await Notification.create({
    userId,
    ...notification,
    sentAt: new Date(),
    read: false
  });
}
```

## Analytics

### События для Отслеживания

```javascript
// Отправка уведомлений
analytics.track('notification_sent', {
  type: 'reminder',
  category: 'food_logging',
  userId: user.id
});

// Взаимодействия
analytics.track('notification_opened', {
  type: 'reminder',
  timeFromSent: 120 // секунд
});

analytics.track('notification_dismissed');
analytics.track('notification_action_clicked', { action: 'add_food' });
```

### Ключевые Метрики
- **Open rate**: % открытых уведомлений
- **CTR**: Click-through rate (клик по действию)
- **Dismiss rate**: % закрытых без действия
- **Opt-out rate**: % отключающих уведомления
- **Engagement lift**: Увеличение активности после уведомления

### Оптимизация
```
A/B тестирование:
- Текст уведомлений
- Время отправки
- Частота
- Эмодзи vs без эмодзи
- Персонализация (имя) vs общее обращение
```

## Best Practices

### DO (Делать)
- ✅ Персонализировать текст
- ✅ Добавлять actionable кнопки
- ✅ Уважать тихие часы
- ✅ Группировать похожие уведомления
- ✅ Давать контроль пользователю (настройки)

### DON'T (Не делать)
- ❌ Спамить уведомлениями
- ❌ Отправлять маркетинг без opt-in
- ❌ Использовать clickbait заголовки
- ❌ Игнорировать настройки пользователя
- ❌ Отправлять ночью (кроме критических)
