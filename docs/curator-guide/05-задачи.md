# Задачи

Задачи -- это конкретные действия, которые ты назначаешь клиенту. Клиент видит задачи в своём дашборде и отмечает выполнение. Ты отслеживаешь прогресс в карточке клиента.

---

## Где находится

Карточка клиента → вкладка **"Задачи"** (`/curator/clients/:id?tab=tasks`).

---

## Список задач

Задачи отображаются с фильтрами:

| Фильтр | Что показывает |
|--------|---------------|
| **Активные** | Задачи в работе (по умолчанию) |
| **Завершённые** | Выполненные клиентом |
| **Просроченные** | С пропущенным дедлайном |

Каждая задача в списке содержит:
- Название и описание
- Тип (иконка)
- Дедлайн
- Статус
- Для повторяющихся -- прогресс (например, "3/7 дней")

---

## Создание задачи

Нажми кнопку **"+"** (круглая кнопка в правом нижнем углу экрана).

| Поле | Описание | Обязательно |
|------|----------|-------------|
| Название | Краткое описание задачи (до 200 символов) | Да |
| Тип | Категория задачи | Да |
| Описание | Подробное описание, инструкции | Нет |
| Дедлайн | Крайний срок выполнения | Да |
| Повторяемость | Разовая, ежедневная или еженедельная | Да |
| Дни недели | Для еженедельных -- выбор конкретных дней | Нет |

### Типы задач

| Тип | Иконка | Примеры |
|-----|--------|---------|
| **Питание** | Столовые приборы | "Съедать минимум 120г белка в день", "Добавить овощи к каждому приёму пищи" |
| **Тренировка** | Гантеля | "Сделать 3 силовые тренировки на этой неделе", "30 минут кардио" |
| **Привычка** | Звезда | "Пить 8 стаканов воды в день", "Ложиться до 23:00" |
| **Замер** | Линейка | "Взвеситься утром натощак", "Сделать фото прогресса" |

### Повторяемость

| Режим | Описание | Прогресс |
|-------|----------|----------|
| **Разовая** | Одно выполнение до дедлайна | Выполнено / не выполнено |
| **Ежедневная** | Каждый день в течение недели | Прогресс-бар "X/7 дней" |
| **Еженедельная** | В выбранные дни недели | Прогресс-бар "X/N раз" |

---

## Удаление задачи

На карточке задачи есть кнопка удаления. Удалённую задачу нельзя восстановить.

---

## Что видит клиент

В дашборде клиента в разделе "Задачи от куратора":

- Список активных задач с иконками типов
- Кнопка-чекбокс для отметки выполнения
- Дедлайн с цветовой индикацией:
  - Серый -- дедлайн в будущем
  - Жёлтый -- дедлайн сегодня
  - Красный -- просрочена
- Для повторяющихся -- мини прогресс-бар
- Выполненные задачи -- зелёный фон с зачёркнутым текстом

### Механика выполнения

- Клиент нажимает кнопку -- задача отмечается как выполненная (оптимистичное обновление)
- Для повторяющихся задач -- каждый день отмечается отдельно
- Если API-вызов не удался -- отметка откатывается

---

## Рекомендации по постановке задач

- **Конкретность:** "Съедать 120г белка" лучше, чем "Есть больше белка"
- **Измеримость:** задача должна иметь чёткий критерий выполнения
- **Реалистичность:** не ставь 7 задач одновременно -- 2-3 на неделю оптимально
- **Привязка к цели:** каждая задача должна работать на фитнес-цель клиента
- **Используй описание:** объясни, зачем эта задача нужна и как её выполнять
