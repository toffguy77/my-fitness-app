# Бизнес-модель My Fitness App

**Версия документа:** 1.0  
**Дата создания:** Январь 2025  
**Статус:** Актуальная реализация v4.0+

---

## Обзор бизнес-модели

**My Fitness App** использует **Freemium SaaS модель** для монетизации платформы управления питанием. Модель основана на предоставлении базового функционала бесплатно для привлечения пользователей и расширенного функционала через Premium подписку для монетизации.

---

## Freemium SaaS модель

### Концепция

**Freemium** — это бизнес-модель, при которой базовый функционал предоставляется бесплатно, а расширенные возможности доступны за плату. Это позволяет:

1. **Привлечение пользователей** — низкий барьер входа (бесплатно)
2. **Монетизация** — конверсия активных пользователей в платных подписчиков
3. **Масштабируемость** — автоматизация процессов через SaaS

### Структура подписок

#### Free (Бесплатный уровень)
**Целевая аудитория:** Пользователи, которые хотят отслеживать питание самостоятельно

**Доступный функционал:**
- ✅ Регистрация и авторизация
- ✅ Онбординг с расчетом целей питания
- ✅ Ввод данных о питании (КБЖУ)
- ✅ Множественные приемы пищи
- ✅ Отслеживание веса тела
- ✅ Базовый дашборд с сводкой за неделю
- ✅ Валидация данных о питании
- ✅ Блокировка редактирования завершенных дней

**Ограничения:**
- ❌ Нет доступа к тренеру
- ❌ Нет отчетов и аналитики
- ❌ Нет графиков прогресса
- ❌ Нет экспорта данных
- ❌ Нет OCR распознавания
- ❌ Нет чата с тренером
- ❌ Нет системы достижений

**Бизнес-ценность:**
- Привлечение новых пользователей
- Создание привычки использования
- Возможность конверсии в Premium

#### Premium (Платный уровень)
**Целевая аудитория:** Пользователи, которые работают с тренером или нутрициологом

**Доступный функционал:**
- ✅ Все функции Free уровня
- ✅ Работа с тренером (получение заметок и обратной связи)
- ✅ Чат с тренером в реальном времени (WebSocket)
- ✅ Доступ к отчетам и аналитике
- ✅ Графики веса и КБЖУ
- ✅ Экспорт данных (CSV, JSON, PDF)
- ✅ Фильтры и пагинация в отчетах
- ✅ Система достижений и бейджей
- ✅ OCR распознавание этикеток продуктов
- ✅ База продуктов с автозаполнением

**Стоимость:**
- Определяется администратором платформы
- Может быть ежемесячной или годовой подпиской

**Бизнес-ценность:**
- Основной источник дохода (MRR)
- Высокая ценность для пользователей
- Долгосрочные отношения с клиентами

## Роли пользователей

### 1. Клиент (Client)

#### Free клиент
**Описание:** Пользователь с базовым доступом к платформе

**Характеристики:**
- `role = 'client'`
- `subscription_status = 'free'`
- `subscription_tier = 'basic'`
- `coach_id = NULL` (не может иметь тренера)

**Возможности:**
- Использование базового функционала дневника питания
- Самостоятельное отслеживание прогресса
- Без доступа к Premium функциям

**Ограничения:**
- Не может быть назначен тренер (защищено триггером в БД)
- Не может получить доступ к отчетам
- Не может использовать OCR и базу продуктов

#### Premium клиент
**Описание:** Пользователь с полным доступом к платформе и работой с тренером

**Характеристики:**
- `role = 'client'`
- `subscription_status = 'active'`
- `subscription_tier = 'premium'`
- `coach_id != NULL` (назначен тренер)
- `subscription_end_date` — дата окончания подписки

**Возможности:**
- Все функции Free уровня
- Работа с тренером (получение заметок, обратная связь)
- Чат с тренером в реальном времени
- Доступ к отчетам и аналитике
- Графики прогресса
- Экспорт данных
- OCR распознавание
- Система достижений

**Ограничения:**
- Подписка может истечь (автоматическая деактивация)
- При истечении подписки теряется доступ к Premium функциям

**Жизненный цикл подписки:**
1. **Активная подписка** (`subscription_status = 'active'`)
   - Полный доступ ко всем функциям
   - Работа с тренером
   - Все Premium возможности

2. **Истекающая подписка** (за 3 дня до окончания)
   - Уведомление о скором истечении
   - Предложение продлить подписку
   - Доступ к функциям сохраняется

3. **Истекшая подписка** (`subscription_end_date < NOW()`)
   - Автоматическая деактивация (`subscription_status = 'cancelled'`)
   - Уведомление об истечении
   - Блокировка Premium функций
   - Возврат к Free уровню

### 2. Тренер (Coach)

**Описание:** Профессионал, работающий с Premium клиентами

**Характеристики:**
- `role = 'curator'`
- Не имеет подписки (роль не требует подписки)
- Может иметь множество клиентов

**Возможности:**
- Кабинет тренера с списком клиентов
- Traffic Light System v2 для приоритизации работы
- Просмотр дашборда клиента
- Редактирование целей питания клиента
- Сохранение заметок для клиента по датам
- Чат с клиентами в реальном времени
- Умная отправка уведомлений (проверка настроек клиента)
- Генерация инвайт-кодов для привлечения клиентов
- Статистика использования инвайт-кодов

**Ограничения:**
- Может работать только с Premium клиентами
- Не может видеть данные Free клиентов
- Не может назначать себя клиентам (только Super Admin)

**Бизнес-ценность:**
- Ключевой элемент Premium подписки
- Обеспечивает ценность для Premium клиентов
- Помогает удерживать клиентов

### 3. Супер-администратор (Super Admin)

**Описание:** Администратор платформы с полным доступом

**Характеристики:**
- `role = 'super_admin'`
- Не имеет подписки (роль не требует подписки)
- Полный доступ ко всем данным

**Возможности:**
- Управление пользователями
- Назначение и изменение ролей
- Управление подписками (активация, деактивация)
- Назначение тренеров клиентам
- Просмотр всех данных системы
- Управление инвайт-кодами

**Ограничения:**
- Только для администраторов платформы
- Не предназначен для обычных пользователей

**Бизнес-ценность:**
- Управление платформой
- Контроль качества сервиса
- Поддержка пользователей

## Монетизация

### Источники дохода

#### 1. Premium подписки
**Основной источник дохода**

- Ежемесячные или годовые подписки
- Автоматическое управление жизненным циклом
- Уведомления о истекающих подписках
- Автоматическая деактивация истекших подписок

**Метрики:**
- MRR (Monthly Recurring Revenue) — ежемесячный регулярный доход
- ARR (Annual Recurring Revenue) — годовой регулярный доход
- Churn rate — процент оттока пользователей
- LTV (Lifetime Value) — ценность клиента за весь период

#### 2. Потенциальные источники (будущее)
- Комиссии с тренеров (если будет реализовано)
- Реклама (для Free пользователей)
- Премиум функции для тренеров

### Стратегия ценообразования

**Текущая модель:**
- Free: Бесплатно (навсегда)
- Premium: Определяется администратором

**Рекомендации:**
- Конкурентная цена на рынке
- Четкое ценностное предложение
- Прозрачность цены и функций

## Ценностное предложение

### Для Free клиентов

**Проблема:**
- Нужен удобный инструмент для отслеживания питания
- Excel и блокноты неудобны
- Нет автоматической аналитики

**Решение:**
- Цифровой дневник питания
- Автоматические расчеты целей
- Визуализация прогресса
- Мобильный доступ (PWA)

**Результат:**
- Удобство использования
- Мотивация к отслеживанию
- Возможность конверсии в Premium

### Для Premium клиентов

**Проблема:**
- Нужна работа с тренером
- Нужна обратная связь и поддержка
- Нужна детальная аналитика

**Решение:**
- Работа с тренером (заметки, обратная связь)
- Чат в реальном времени
- Детальные отчеты и графики
- OCR для быстрого ввода
- Система достижений

**Результат:**
- Эффективная работа с тренером
- Мотивация и поддержка
- Достижение целей

### Для тренеров

**Проблема:**
- Нужен инструмент для мониторинга клиентов
- Нужна возможность давать обратную связь
- Нужна приоритизация работы

**Решение:**
- Кабинет тренера с списком клиентов
- Traffic Light System для приоритизации
- Сохранение заметок по датам
- Чат с клиентами
- Редактирование целей питания

**Результат:**
- Эффективная работа с клиентами
- Экономия времени
- Улучшение результатов клиентов

## Ключевые бизнес-метрики

### Пользовательские метрики

#### Активные пользователи
- **DAU (Daily Active Users)** — ежедневно активные пользователи
- **MAU (Monthly Active Users)** — ежемесячно активные пользователи
- **Retention Rate** — процент удержания пользователей

#### Вовлеченность
- **Days with Data** — среднее количество дней с данными в месяц
- **Meals per Day** — среднее количество приемов пищи в день
- **Check-in Rate** — процент завершенных дней

### Бизнес-метрики

#### Доход
- **MRR (Monthly Recurring Revenue)** — ежемесячный регулярный доход
- **ARR (Annual Recurring Revenue)** — годовой регулярный доход
- **Average Revenue Per User (ARPU)** — средний доход с пользователя

#### Конверсия
- **Free to Premium Conversion Rate** — процент конверсии Free → Premium
- **Trial to Paid Conversion Rate** — процент конверсии пробного периода → платная подписка

#### Отток
- **Churn Rate** — процент оттока пользователей
- **Revenue Churn** — отток дохода
- **Customer Lifetime Value (LTV)** — ценность клиента за весь период

#### Привлечение
- **Customer Acquisition Cost (CAC)** — стоимость привлечения клиента
- **LTV/CAC Ratio** — соотношение ценности клиента к стоимости привлечения
- **Invite Code Usage** — использование инвайт-кодов тренерами

### Технические метрики

#### Производительность
- **Page Load Time** — время загрузки страниц
- **API Response Time** — время ответа API
- **Database Query Time** — время выполнения запросов к БД

#### Надежность
- **Uptime** — доступность сервиса
- **Error Rate** — процент ошибок
- **Success Rate** — процент успешных операций

## Риски и митигация

### Риск 1: Низкая конверсия Free → Premium
**Митигация:**
- Улучшение ценностного предложения
- Оптимизация процесса конверсии
- A/B тестирование

### Риск 2: Высокий отток пользователей
**Митигация:**
- Улучшение качества продукта
- Эффективная работа с тренером
- Мотивация через достижения

### Риск 3: Зависимость от тренеров
**Митигация:**
- Разнообразие функций (не только работа с тренером)
- Самостоятельное использование (Free уровень)
- Расширение функционала
