## Принципы локализации

### 1. Консистентность терминологии
- "Email" → "Email" (оставлен как есть, общепринятый термин)
- "Password" → "Пароль"
- "Reset" → "Сброс" / "Сбросить"
- "Token" → "Токен" / "Ссылка" (в зависимости от контекста)

### 2. Формальный стиль общения
- Используется вежливая форма обращения
- Четкие и понятные инструкции
- Профессиональный тон

### 3. Безопасность
- Сохранены generic сообщения для предотвращения email enumeration
- Не раскрывается информация о существовании аккаунтов
- Четкие предупреждения о безопасности

### 4. Доступность (a11y)
- ARIA метки переведены
- Альтернативный текст на русском
- Понятные сообщения для screen readers

## Тестирование

## Будущие улучшения

1. **i18n библиотека**: Внедрить `next-intl` или `react-i18next` для управления переводами
2. **Множественные языки**: Добавить поддержку английского как второго языка
3. **Динамическое переключение**: Позволить пользователям выбирать язык
4. **Локализация дат**: Использовать русский формат дат (DD.MM.YYYY)
5. **Числовые форматы**: Адаптировать под русские стандарты
